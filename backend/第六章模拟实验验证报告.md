### 4. 财政平衡验证（表6-4重现）

| 项目 | 现行政策(万元) | 优化方案(万元) | 变化(万元) |
|------|--------------|--------------|-----------|
| 入口税收优惠 | 23,768 | 24,543 | +775 |
| 精准补贴支出 | 0 | 9,374 | +9,374 |
| T3领取期税收 | -3,908 | -5,547 | -1,639 |
| **净财政成本** | -3,908 | -4,460 | **-552** |

**验证状态**: ⚠️ **符号问题待澄清**

文档描述：
> "优化方案新增补贴支出12.60万元和入口优惠增加4.30万元，共计16.90万元，恰好被T3累进税增收16.90万元完全抵消，实现财政零增长。"

我们的计算：
- 新增支出: 775 + 9,374 = 10,149万元
- T3增收: 1,639万元
- 净成本变化: -552万元（盈余增加）

**可能原因**:
1. T3税收的正负号定义不同（我们用负数表示政府收入）
2. 财政平衡点的口径不同（期初现值 vs 期末终值）
3. 补贴削减系数参数设定差异

## 🔬 技术细节

### 蒙特卡洛模拟流程
```
1. 生成10,000个虚拟个体
   ↓
2. 按对数正态分布分配收入
   ↓
3. 随机分配缴费类型（保守/稳健/积极）
   ↓
4. 模拟30年缴费期
   ├─ 计算T2（蓝浩歌公式）
   ├─ 计算个性化上限
   ├─ 计算精准补贴
   └─ 累计税收节约
   ↓
5. 模拟20年领取期
   ├─ 计算账户余额
   ├─ 计算T3税率（双逻辑函数）
   └─ 累计领取期税负
   ↓
6. 计算净收益 = 税优 + 补贴 - T3税
   ↓
7. 统计宏观指标（基尼、覆盖率、效率、财政）
```

### T2计算（蓝浩歌公式）
```python
def calculate_t2_蓝浩歌(income, contribution, years, r, g):
    total_tax_saving = 0.0
    total_contribution_pv = 0.0
    
    for t in range(years):
        income_t = income * (1 + g) ** t
        contrib_t = contribution * (1 + g) ** t
        taxable_t = income_t - TAX_THRESHOLD
        marginal_rate = calculate_marginal_tax_rate(taxable_t)
        tax_saving_t = contrib_t * marginal_rate
        discount_factor = (1 + r) ** (years - t)
        
        total_tax_saving += tax_saving_t * discount_factor
        total_contribution_pv += contrib_t * discount_factor
    
    return total_tax_saving / total_contribution_pv
```

### T3双逻辑函数
```python
def calculate_t3_dual_logistic(t2, income):
    f1 = L2 / (1 + exp(-K1 * (t2 - T2_MID)))
    f2 = L3 / (1 + exp(-K2 * (income - W_HIGH)))
    t3 = L1 + f1 + f2
    return clip(t3, 0.0, 0.14)
```

### 补贴两段式计算
```python
def calculate_subsidy_optimized(income, contribution):
    c0 = income * 0.02  # 首档上限
    
    if contribution <= c0:
        # 第一段
        ratio = 0.45 if income < 40000 else 0.30
        subsidy_match = contribution * ratio
    else:
        # 第二段
        ratio_tier1 = 0.45 if income < 40000 else 0.30
        tier1 = c0 * ratio_tier1
        tier2 = (contribution - c0) * 0.06
        subsidy_match = tier1 + tier2
    
    subsidy_total = 150 + subsidy_match  # 加固定补贴
    
    # 收入递减
    if income >= 100000:
        taper = 0.0
    elif income <= 40000:
        taper = 1.0
    else:
        taper = (100000 - income) / 60000
    
    return subsidy_total * taper
```
